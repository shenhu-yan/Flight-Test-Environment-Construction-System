# 飞行试验环境系统用户手册

## 系统概述

飞行试验环境系统是一个专为强化学习算法训练设计的专业化试验环境平台，实现了飞行试验环境的全流程自动化管理。

### 核心功能
- 环境生成：批量生成多样化飞行试验环境
- 环境调整：实时动态调整环境参数
- 智能优化：基于强化学习的环境质量评估与优化
- 模型管理：多类型模型的全生命周期管理
- 用户交互：直观易用的可视化界面

## 系统登录

1. 打开浏览器，访问前端服务地址（默认：http://localhost:3000）
2. 输入用户名和密码（默认用户见部署指南）
3. 点击「登录」按钮进入系统
4. 登录成功后系统会自动保存会话，刷新页面无需重新登录

> 注意：系统使用 JWT 令牌进行身份认证，令牌有效期为 15 分钟。过期后需重新登录。

## 功能模块使用指南

### 1. 仪表盘

**功能**：展示系统概览和关键指标

**操作**：
- 查看系统状态、用户数量、项目数量、环境数量等统计信息
- 查看最近操作记录
- 快速访问常用功能（环境生成、环境调整、环境优化、模型管理）

### 2. 项目管理

**功能**：创建和管理项目

**操作**：
- **创建项目**：点击「创建项目」按钮，填写项目名称、项目ID和描述，点击「创建」确认
- **编辑项目**：点击项目列表中的「编辑」按钮修改项目名称和描述
- **删除项目**：点击项目列表中的「删除」按钮（谨慎操作，删除后不可恢复）
- **查看项目详情**：在环境管理或模型管理页面通过项目下拉菜单筛选查看

> 注意：项目ID创建后不可修改，请确保填写正确的项目ID。

### 3. 环境管理

**功能**：管理所有飞行试验环境

**操作**：
- **筛选环境**：通过项目下拉菜单筛选特定项目的环境
- **查看环境**：查看环境列表中的基本信息（名称、ID、项目、状态、创建时间）
- **调整环境**：点击「调整」按钮进入环境调整页面
- **优化环境**：点击「优化」按钮进入环境优化页面
- **删除环境**：点击「删除」按钮删除环境

### 4. 环境生成

**功能**：创建新的飞行试验环境

**操作**：
1. **选择生成方式**：
   - 自定义配置：手动设置所有参数
   - 模板配置：选择预设模板快速生成

2. **填写基本信息**：
   - 项目：选择环境所属项目（必填）
   - 环境名称：输入环境名称
   - 批量生成数量：设置生成数量（1-10）

3. **配置环境参数**：
   - **场景配置**：地形（平坦/山地/城市/海洋）、气象（晴朗/有风/大雨/雪）、障碍物（建筑/树木/电线/鸟类）
   - **物理规则**：飞行力学（基础/中级/高级）、空气动力学（基础/详细/精确）、碰撞检测（启用/禁用）
   - **奖励函数**：奖励值、惩罚规则（碰撞/越界/超时/燃料耗尽）、目标阈值

4. **点击「生成环境」按钮**：
   - 系统开始生成环境
   - 生成完成后弹出成功提示

> 注意：必须先选择项目才能生成环境。如未选择项目，系统会提示"请选择项目"。

**模板管理**：
- 点击「模板管理」标签页
- **创建模板**：点击「创建模板」按钮，填写模板信息（仅管理员和配置人员可用）
- **编辑模板**：点击模板列表中的「编辑」按钮（仅管理员和配置人员可用）
- **删除模板**：点击模板列表中的「删除」按钮（仅管理员和配置人员可用）

### 5. 环境调整

**功能**：实时调整环境参数

**操作**：
1. **选择环境**：从下拉菜单选择要调整的环境

2. **自动调整**：
   - 进入「自动调整」标签页
   - 输入性能指标（奖励值、成功率、收敛速度、泛化能力）
   - 点击「自动调整」按钮
   - 系统根据预设策略自动调整环境
   - 调整成功后弹出提示

3. **手动调整**：
   - 进入「手动调整」标签页
   - 输入调整原因
   - 修改环境参数（修改不会影响原始配置）
   - 点击「手动调整」按钮

4. **查看调整历史**：
   - 进入「调整历史」标签页
   - 查看所有调整记录（调整者、触发方式、参数、原因、时间）
   - 点击「回滚」按钮回滚到指定版本

> 注意：手动调整时修改的参数是深拷贝，不会影响当前环境的实际配置，确认提交后才会生效。

### 6. 环境优化

**功能**：评估和优化环境质量

**操作**：
1. **选择环境**：从下拉菜单选择要优化的环境

2. **环境评估**：
   - 进入「环境评估」标签页
   - 输入性能指标
   - 点击「评估环境」按钮
   - 查看评估结果（多样性、挑战性、真实性、有效性评分）和优化建议

3. **环境优化**：
   - 进入「环境优化」标签页
   - 输入性能指标
   - 设置自定义优化目标（可选）
   - 点击「优化环境」按钮
   - 查看优化结果和改进幅度

4. **批量优化**：
   - 进入「批量优化」标签页
   - 勾选多个环境
   - 设置优化目标
   - 点击「批量优化」按钮

### 7. 模型管理

**功能**：管理各种类型的模型

**操作**：
1. **上传模型**：
   - 点击「上传模型」按钮
   - 填写模型信息（项目、任务ID、类型、版本、描述）
   - 选择模型文件
   - 点击「上传」按钮

2. **管理模型**：
   - **查看模型**：查看模型列表中的基本信息
   - **下载模型**：点击「下载」按钮下载模型文件（需登录认证）
   - **删除模型**：点击「删除」按钮删除模型（需确认）

3. **筛选模型**：
   - 通过项目下拉菜单筛选
   - 通过模型类型下拉菜单筛选（场景模型/物理模型/算法模型）

### 8. 用户管理（仅管理员）

**功能**：管理系统用户

**操作**：
- 查看所有用户列表（用户名、角色、创建时间）
- 创建新用户（设置用户名、密码、角色）
- 删除用户

## 角色权限

| 角色 | 权限范围 | 可访问功能 |
|------|----------|------------|
| 系统管理员 (admin) | 所有权限 | 全部功能，包括用户管理 |
| 配置人员 (config) | 环境配置权限 | 环境生成、调整、优化、模板管理、模型管理 |
| 研发人员 (dev) | 算法研发权限 | 环境生成、调整、优化、模型上传/下载/删除 |
| 普通用户 (user) | 查看权限 | 仪表盘、环境查看、数据查看、结果导出 |

> 注意：研发人员 (dev) 可以使用环境生成、调整、优化和模型管理功能，但不能管理模板和用户。

## 常见操作流程

### 1. 完整的环境生命周期

1. **创建项目**：在「项目管理」页面创建新项目
2. **生成环境**：在「环境生成」页面为项目创建环境
3. **训练算法**：使用生成的环境训练强化学习算法
4. **调整环境**：根据算法性能调整环境参数
5. **优化环境**：评估并优化环境质量
6. **迭代改进**：重复步骤3-5直至达到目标性能

### 2. 模型管理流程

1. **上传模型**：在「模型管理」页面上传模型文件
2. **版本管理**：系统自动记录模型版本
3. **模型使用**：在环境生成和优化中使用模型
4. **模型更新**：上传新版本模型

| 模板类型 | 复杂度 | 试验类型 | 适用场景 |
|---------|--------|----------|----------|
| 固定翼 | 基础 | 机动飞行 | 新手训练 |
| 固定翼 | 中等 | 故障模拟 | 算法测试 |
| 固定翼 | 高级 | 环境适应 | 极限测试 |
| 旋翼 | 基础 | 机动飞行 | 新手训练 |
| 旋翼 | 中等 | 故障模拟 | 算法测试 |
| 旋翼 | 高级 | 环境适应 | 极限测试 |
| 无人机 | 基础 | 机动飞行 | 新手训练 |
| 无人机 | 中等 | 故障模拟 | 算法测试 |

### 2. 调整策略配置

系统内置以下调整策略：

| 性能情况 | 调整策略 |
|---------|----------|
| 算法收敛过快/奖励值过高 | 增加环境复杂度，优化奖励机制 |
| 算法收敛过慢/成功率过低 | 降低环境复杂度，调整奖励机制 |
| 算法泛化能力不足 | 随机引入新场景元素 |

## 故障排除

### 1. 环境生成失败
- 检查是否已选择项目
- 检查输入参数是否合法
- 检查项目是否存在
- 查看弹出的错误提示获取详细信息

### 2. 环境调整失败
- 检查环境状态是否正常
- 检查调整参数是否合法
- 检查用户权限是否足够

### 3. 模型上传失败
- 检查文件格式是否支持
- 检查文件大小是否超限
- 检查项目权限

### 4. 登录后操作无效果
- 检查 JWT 令牌是否过期（有效期15分钟）
- 尝试重新登录
- 检查浏览器控制台是否有错误信息

### 5. 页面显示异常
- 清除浏览器缓存后重试
- 检查网络连接
- 确保后端服务正常运行
